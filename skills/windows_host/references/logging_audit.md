# Windows Logging and Audit Coverage

## Checks
1. Advanced audit policy subcategories enabled for logon, account management, policy change, and object access.
2. PowerShell script block logging, module logging, and transcription enabled.
3. Event log sizes and retention configured to prevent overwrite.
4. Security event forwarding or SIEM integration configured where required.
5. Sysmon presence and configuration baseline documented if in use.

## Evidence Capture
1. Audit policy export and PowerShell logging configuration evidence.
2. Event log size/retention settings and forwarding destinations.
